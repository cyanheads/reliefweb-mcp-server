# 0.1.4 — 2026-05-24

## Changed

- **`ReliefWebService`** — static `import { serviceUnavailable }` replaces two dynamic `await import()` calls in the hot path (`post()` and `get()`).
- **`post()` / `get()`** — spurious `async` removed; both are synchronous wrappers returning a `Promise`.
- **`makeDateFilter`** return type tightened to `FilterCondition`, eliminating three `as FilterCondition` casts at call sites.
- **`normalizeLinks` / `normalizeDatedLinks`** — extracted shared helpers unifying two copy-pasted normalization blocks in `normalizeDisasterDetail` and `normalizeCountryDetail`.
- **Status value handling** — dead-code path removed; `values[0] as string` replaces the unreachable `?? values` fallback.
- **`@cyanheads/mcp-ts-core`** ^0.9.7 → ^0.9.9.
- **`manifest.json` / README description** — aligned to "reports, disasters, jobs, training, and country profiles" (was "reports, disasters, jobs, and training from OCHA").
- **Skills** — `git-wrapup` v1.0 added; `release-and-publish` updated to v2.6.
- **`CLAUDE.md` / `AGENTS.md`** — "Close the loop on issues" rule synced from framework 0.9.9; framework version reference updated to `^0.9.9`.

## Fixed

- **`reliefweb://disasters/{id}`** — validation error for non-integer `id` now throws `validationError` (was `notFound`).
- **`reliefweb://reports/{id}`** — validation error for non-integer `id` now throws `validationError` (was `notFound`).
- **`reliefweb://countries/{iso3}`** — invalid ISO3 format now throws `validationError` (was `notFound`).
