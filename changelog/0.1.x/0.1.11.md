# 0.1.11 — 2026-06-15

## Added

- **`upstream_error` typed contract** on the six search/list tools (`reliefweb_search_reports`, `reliefweb_search_disasters`, `reliefweb_search_jobs`, `reliefweb_search_training`, `reliefweb_list_countries`, `reliefweb_list_sources`) — each declares `errors: [{ reason: 'upstream_error', code: ServiceUnavailable, … }]` and wraps the service call in a `.catch` that re-throws via `ctx.fail`, so a ReliefWeb API failure carries `data.reason` for agent routing instead of message-text parsing. ([#6](https://github.com/cyanheads/reliefweb-mcp-server/issues/6))
- **`appliedFilters` output field** on the four search tools — echoes the resolved filter set the query actually ran with (normalized country, resolved `sort`/`preset` defaults, `limit`/`offset`), rendered into `content[]` so content[]-only clients see how their inputs were interpreted. ([#7](https://github.com/cyanheads/reliefweb-mcp-server/issues/7))
- **`notice` enrichment** on `reliefweb_list_countries` and `reliefweb_list_sources` — empty filtered results now return a recovery hint echoing the active filter (`crisis_only`, `text`, `type`) and how to broaden. ([#7](https://github.com/cyanheads/reliefweb-mcp-server/issues/7))

## Changed

- **Empty-result notices** on the four search tools now echo every active filter — `search_reports` adds `disaster_id`/`language`/`source`/`date_from`/`date_to`; `search_disasters` adds `glide`/`date_from`/`date_to`; `search_jobs` adds `source`/`theme`/`experience`; `search_training` adds `source`/`career_category`/`language`/`date_start_to`. ([#7](https://github.com/cyanheads/reliefweb-mcp-server/issues/7))
- **`release:github` script** invokes `bun run scripts/release-github.ts` instead of `tsx`; the `tsx` devDependency is removed.
