# 0.1.12 — 2026-06-20

Framework-maintenance release. No tool, resource, or server behavior changes.

## Added

- **`scripts/check-dependency-specifiers.ts`** — new `devcheck` step (`Dependency Specifiers`, `--no-dep-specifiers`) that hard-fails on floating specifiers (`latest`, `*`, pre-release dist-tags) in `package.json`'s dependency sections and `bun.lock`'s `workspaces` map. Catches a `bun update --latest` writing a literal `latest` dist-tag into the lock and re-resolving a held dependency past its range. ([cyanheads/mcp-ts-core#246](https://github.com/cyanheads/mcp-ts-core/issues/246))
- **Plugin marketplace manifest checks in `lint:packaging`** — validates `.claude-plugin/plugin.json`, `.codex-plugin/plugin.json`, and `.codex-plugin/mcp.json`: non-empty descriptions, unscoped machine name in display fields, full `package.json` name in the `npx -y` install arg. Gated by the new `devcheck.config.json` `packaging.pluginManifests` flag (default on); the `devcheck` Packaging gate now runs when any plugin manifest is present. ([cyanheads/mcp-ts-core#240](https://github.com/cyanheads/mcp-ts-core/issues/240))

## Changed

- **`.codex-plugin/plugin.json`** — populated the empty `interface.longDescription` (now surfaced by the plugin-manifest lint).
- **Vendored framework scripts** — `build-changelog.ts`, `check-framework-antipatterns.ts`, `check-skill-versions.ts`, `devcheck.ts`, and `lint-packaging.ts` re-synced from `@cyanheads/mcp-ts-core` to pick up fresh-scaffold guards (changelog-sync and git-dependent checks skip when `.git` is absent), the `check-skill-versions` worktree-deletion guard, and the new packaging checks. ([cyanheads/mcp-ts-core#237](https://github.com/cyanheads/mcp-ts-core/issues/237), [#242](https://github.com/cyanheads/mcp-ts-core/issues/242), [#243](https://github.com/cyanheads/mcp-ts-core/issues/243))
- **Vendored framework skills** — re-synced `add-tool`, `api-canvas`, `api-config`, `api-context`, `api-telemetry`, `git-wrapup`, `orchestrations`, and `polish-docs-meta` to their upstream versions (including the `ctx.content` collector docs in `api-context` and corrected `OTEL_SERVICE_NAME` seeding in `api-config`/`api-telemetry`).
